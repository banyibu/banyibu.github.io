Docker-基本操作

计算机技术,Docker

概述

Docker的操作，这里包括，镜像、容器、资源控制。几方面的操作。

部署KVM虚拟化平台

KVM(Kernel Virtual Machine)虚拟化技术，KVM是linux内核的一个模块。
部署虚拟化环境、创建虚拟机实例、虚拟机的基本管理。
KVM优化主要针对CPU、内存、I/O这几个方面。
1.其中CPU总数不要超过物理机CPU的总数目。
2.内存可以使用KSM(Kernel Samepage Merging,相同页合并)优化；可以使用大页，此时页表大小会降低，提高CPU效率。
3.对磁盘I/O控制有两种方式：修改整体中的读写权重(100~1000)、限制具体的I/O

Docker概念

概念：镜像、容器、仓库；安装Docker以及围绕镜像和容器的具体操作。
Docker容器除了运行其中的应用之外，基本不消耗额外的系统资源。
Docker操作方便，还可以通过Dockerfile配置文件支持灵活的自动化创建和部署。
镜像:Docker 的镜像是创建容器的基础，类似虚拟机的快照，可以理解为一个面向Docker容器引擎的只读模板。
容器:Docker 的容器是从镜像创建的运行实例，它可以被启动、停止和删除。
仓库:用来保存Docker镜像的地方，自己创建的镜像可以通过push命令将其上传到公共仓库或私有仓库。

Docker的基本操作

Docker安装完成后有Docker服务端和客户端。服务端是一个服务进程，管理着所有的容器。Docker客户端扮演者服务端的远程控制器，可以用来控制Docker的服务进程。

注意，Docker运行容器前需要本地存在对应的镜像，不存在时，Docker会从默认的镜像仓库中下载。
2.Docker镜像操作

  1. 搜索镜像：docker search 指定关键词 ,返回的镜像按照星级拍卖，可以使用参数-s或--stars=x显示指定星级以上的镜像；其中官方创建的镜像使用单个单词命名，带前缀的为用户创建；
  2. 获取镜像：docker pull 仓库名[:标签]

    镜像文件由若干层组成，被称之为AUFS(联合文件系统)，是实现增量保存与更新的基础。
    若要从其他注册服务器下载镜像，需要在仓库名称前指定完整的仓库注册服务器地址。
  3. 查看镜像信息：docker images查看下载到本地的镜像。

    根据镜像的唯一标识ID，获取镜像详细信息：docker inspect 镜像ID号

    为镜像添加新标签：docker tag 名称:[标签] 新名称:[新标签]
  4. 删除镜像：docker rmi 仓库名称:标签或docker rmi 镜像ID号

    注意：当一个镜像有多个标签时，删除时只会删除该镜像指定的标签。
    当使用ID号删除时，就需要保证该镜像没有被容器使用。
  5. 载入镜像和存出镜像：
    存出镜像：docker save -o 存储文件名 存出的镜像
    载入镜像：docker load<存出的文件或docker --input 存出的文件
  6. 上传镜像：docker push 仓库名称:标签

    3.Docker容器操作
    容器是镜像的一个运行实例，是独立运行的一个或一组应用以及它们所必须的运行环境，包括文件系统、系统类库、shell环境等。
  7. 容器的创建与启动：
    容器的创建就是将镜像加载到容器的过程，新创建的容器默认处于停止状态，不运行任何程序，需要在其中发起一个进程来启动容器，若容器中最后一个进程结束，则该容器也会完全停止。停止的容器可以启动并保留原来的修改。
    创建容器命令：docker create [选项] 镜像 运行的程序；常用选项：-i 让容器的输入保持打开；-t 让Docker分配一个伪终端

    查看所有容器的运行状态的命令：docker ps; 添加-a选项可以列出系统最近一次启动的容器。

    启动停止状态的容器：docker start 容器名称/ID

    创建并启动容器，可以直接使用docker run 镜像 执行命令；若添加-d选项，则容器在后台持续运行这个容器，但是注意容器所运行的程序不能结束。需要注意的是，命令执行完成后，镜像会自动退出。
  8. 容器的运行与终止：docker stop 容器的ID/名称
  9. 容器的进入：docker exec -it 容器名称/ID /bin/bash

    在容器终端，通过exit退出容器。
  10. 容器的导出导入：容器导出：docker export 容器ID/名称 > 文件名；容器导入：docker import命令，具体格式为：cat 导出文件名 | docker import 生成的镜像名称:标签
  11. 容器的删除：docker rm 容器ID/名称，若要强制删除一个正在运行的容器，可以添加-f选项强制删除。
    Docker默认的存储目录在/var/lib/docker，Docker的镜像、容器、日志等内容全部都存储在此。
    4.Docker资源控制
    Cgroup是Control group的简写，是linux内核提供的一种限制使用物理资源的机制。
