golang第三章：基本数据

计算机基础,计算机语言,go程序设计语言

概述

go的数据类型分为四大类：基础类型、聚合类下、引用类型和接口类型。
其中引用类型包括：指针、slice、map、函数以及通道。

整数

有符号整数：int8、int16、int32、int64
无符号整数：uint8、uint16、uint32、uint64
int，uint，其大小和硬件及编译器有关。
rune类型和int32是同义词，byte类型和uint是同义词。
一种无符号整数：uintptr
Go的二元操作：* / % << >> & &^(位清空) + - | ^ == != < <= > >= && ||
Go中取模余数的正负号总是与被除数保持一致。
无符号数往往只用于位运算和特定的算数运算。

浮点数

Go有float32和float64两种浮点数类型。
math包给出了常量math.MaxFloat32=3.4e38，math.MaxFloat64=1.8e308
其中float32有效位是6，float64有效位是15位。
格式化输出时，%g保存足够的精度，并自动选择简洁的表示方式；%e有指数的形式输出，%f无指数的形式输出；
math.IsNaN()判读其参数是否是非数值；math.NaN()返回非数值(NaN)；注意：NaN不能用来比较；

复数

Go具有两种大小的复数，complex64和complex128，二者分别由float32和float64构成。
内置函数complex()、real()、imag()
math/cmplx包提供了复数所需的库函数。
math/big包给出了big.Float类型，但是精度有限；big.Rat类型使用无限精度

布尔值

短路计算方式。
&&优先级高于||

字符串

内置函数len返回字符串字节数。
字符串内的数据不允许修改，所以其子串可以和原串共享内存。

字符串常量

Go的源文件总是按UTF-8编码，并且习惯上Go的字符串会按UTF-8解读。
转义字符，除了包含特殊字符外，也可以包含十六进制或八进制的转义。\xhh \ooo, 其中，八进制不能超过\377
原生的字符串字面量：`……`的形式，其内容与字面写法一致，只有回车符会被删除，但是换行符保留。

Unicode

Unicode码点数，在Go中叫文字符号(rune)。为int32的别名。UTF-32,每个字符占32位，是等长编码。

UTF-8

UTF-8以字节为单位对Unicode做变长编码。变长编码的字符串无法按照下标直接访问第n个字符。
UTF-8编码本身不会嵌入NULL字节，所以某些程序可以使用NUL标记字符串结尾。
Go的源文件总是以UTF-8编码的，Go程序要操作的文本字符串也优先采用UTF-8编码。
unicode包具备对单个文字符号的函数，unicode/utf8包按UTF-8编码和解码文字符号的函数。
Go中可以使用码点的值来指明Unicode字符。\uhhhh表示16位码点的值，\uhhhhhhhh表示32位码点的值。
Go中范围for也可以用于按照UTF-8编码的字符串。
注意len返回字符串字节数，utf8.RuneCountInString()返回码点或文字符号的个数。

若UTF-8解读文本时，读入一个不合理的字节，那么会产生一个Unicode字符‘\ufffd’是一个黑色的六角形或类似钻石的形状，内部有一个问好。

字符串和字节slice

4个标准包对字符串操作：bytes、strings、strconv、unicode
由于字符串不可变，按照增量的方式构建字符串会导致多次内存分配和复制，这时使用bytes.Buffer类型效率更高。
path包和path/filepath包处理文件路径更适合。
字符串可以和字节slice相互转换，字符串创建后就不能改变了，但是字节slice可以相互转换。

字符串和数字的相互转化

整数转字符串，fmt.Sprintf或strconv.Itoa

常量

编译时就计算出表达式的值。
对于常量操作数，所有数学运算、逻辑运算和比较运算的结果依然是常量。

常量生成器itoa

常量声明中itoa从0开始取值，逐项加1。

无类型常量

虽然常量可以是任何任何基本的数据类型，但是许多常量并不从属某一具体类型。
从属类型待定的常量共有6种：无类型布尔、无类型整数、无类型文字符号、无类型浮点数、无类型复数、无类型字符串。
只有常量才可以使无类型的。
无类型整数可以转换成int，但是其类型是不确定的。
